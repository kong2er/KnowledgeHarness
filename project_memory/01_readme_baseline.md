# KnowledgeHarness README Baseline (Memory Copy)

KnowledgeHarness 是一个基于 Harness Engineering 思想构建的学习资料整理工具。
它以用户上传的资料为核心，通过“资料解析 → 内容切分 → 自动分类 → 阶段总结 → 重点提炼 → 网络补充 → 结果校验”的流程，将零散资料整理为结构化笔记。

## 1. 项目背景

在日常学习中，资料往往来自多个来源，例如：

- 课堂笔记
- PDF 讲义
- PPT 导出内容
- 自己整理的 txt / md
- 截图或拍照图片
- 网络搜索得到的补充资料

这些资料通常存在以下问题：

- 来源分散，难以统一整理
- 内容重复、交叉、混乱
- 缺少清晰的阶段性总结
- 临近复习时难以快速提炼重点
- 外部补充资料缺少有效筛选和统一归档

因此，我设计了 KnowledgeHarness，希望将“资料收集—整理—总结—补充—导出”做成一个有规则、有工具、有验证闭环的 AI 原生工作流。

## 2. 核心目标

KnowledgeHarness 的目标不是做一个普通聊天总结器，而是做一个可执行的学习资料整理流程系统。

系统需要实现以下能力：

1. 读取用户上传的原始资料
2. 将资料切分成独立内容块
3. 对内容块进行自动分类
4. 生成阶段性总结
5. 提炼最终重点记要
6. 通过网络搜索进行有限补充
7. 输出带来源与链接的结构化笔记
8. 对结果进行验证，减少错误归类与信息混淆

## 3. 功能设计

### 3.1 输入支持
系统当前计划支持以下输入类型：

- `.txt`
- `.md`
- `.pdf`
- 图片（后续扩展 OCR）
- 用户手动输入主题关键词
- 用户选择是否开启网络补充

### 3.2 资料解析
系统首先对原始资料进行统一解析，提取纯文本内容，并记录来源信息。

每条内容至少保留以下元数据：

- source_name
- source_type
- chunk_id
- raw_text
- extracted_text

### 3.3 内容切分
系统不会直接对整份长文本做一次性总结，而是先将资料切分为多个语义块，以便后续分类与归并。

### 3.4 自动分类
系统将每个内容块分类到预设类别中，例如：

- 基础概念
- 方法原理
- 应用案例
- 易错点 / 难点
- 扩展阅读
- 待确认

### 3.5 阶段总结
系统分阶段生成总结，而不是只输出最后一次总摘要。

建议分为三阶段：

- 阶段 1：资料概览与主题分布
- 阶段 2：分类后各模块总结
- 阶段 3：重点记要与复习建议

### 3.6 网络补充
系统允许基于当前主题进行有限网络搜索，用于：

- 补定义
- 补背景
- 补官方/高质量资料链接

网络补充必须遵守以下规则：

- 用户资料优先
- 网络搜索只作补充，不替代用户原始内容
- 外部信息必须保留链接
- 外部内容与用户原始资料必须分开展示

### 3.7 输出结果
系统最终输出结构化笔记，至少包括：

- 资料概览
- 分类结果
- 分阶段总结
- 重点记要
- 网络补充资源
- 待复核内容

## 4. Harness Engineering 设计

本项目的重点不只是“调用大模型生成结果”，而是通过 Harness Engineering 思想，将 Agent 的工作流程工程化。

### 4.1 Context Management / Agent Skill
项目通过 `SKILL.md` 约束 Agent 的行为，包括：

- 用户资料优先于网络资料
- 先分类，后总结
- 低置信度内容进入待确认区
- 不允许将冲突内容强行合并
- 外部补充必须附带来源链接

### 4.2 Tool Use / 外部工具调用
系统通过本地工具链完成资料处理，而不是依赖单轮对话直接生成：

- 文件解析器
- PDF 文本提取
- 文本切分器
- 分类器
- 搜索模块
- 校验模块
- 导出模块

### 4.3 Feedback Loop / 验证与反馈闭环
系统将对结果进行自动检查，例如：

- 是否存在大量未分类内容
- 是否有分类冲突
- 是否存在高重复内容
- 是否遗漏主分类
- 外部资源是否带链接
- 输出是否覆盖主要主题

若检查失败，则进入以下流程之一：

- 重新分类
- 重新补充搜索
- 标记为 `review_needed`

## 5. 系统流程

```text
用户上传资料
    ↓
资料解析
    ↓
内容切分
    ↓
自动分类
    ↓
阶段总结
    ↓
重点提炼
    ↓
网络补充
    ↓
结果校验
    ↓
输出结构化笔记
```
