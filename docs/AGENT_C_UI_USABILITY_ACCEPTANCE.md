# AGENT_C_UI_USABILITY_ACCEPTANCE

Last Updated: 2026-04-22

Agent C（增强验收与可用性把关 Agent）负责 UI 可用性验收与回归把关。

## 1. 可用性硬验收项

### 1.1 首屏可理解性

- 首屏必须明确“先导入、再配置、后运行”
- 主按钮必须可见且突出（开始整理/运行流水线）
- 不得让用户在首屏猜测主动作

### 1.2 处理中可感知性

- 提交后按钮必须禁用并显示处理中状态
- 页面必须出现可见运行状态提示
- 右栏必须显示阶段链与统计位

### 1.3 结束后可达性

- 结果区必须可见
- 至少提供：打开结果文件、打开输出目录
- 若结果未生成，按钮必须禁用或明确“未生成”

### 1.4 错误可理解性

- 输入错误返回 400 且保持 UI 存活
- 运行异常返回 500 且保持 UI 存活
- 错误提示必须为用户可读文本，不得仅暴露内部异常名

## 2. 按钮层级验收

- 一级按钮：开始整理（唯一主动作）
- 二级按钮：选择文件/选择文件夹/API 设置/打开结果/打开目录
- 次级动作：重置本次选择/清空列表/删除

验收标准：

- 一级按钮视觉权重大于其他按钮
- 不得将“打开结果”设计成与“开始整理”同等权重

## 3. 3-Agent 边界验收

- Agent A：仅负责输入可见性与输入错误映射
- Agent B：仅负责状态/结果信息架构
- Agent C：仅负责可用性验收与回归

禁止跨界：

- C 不得改写 pipeline 业务语义
- C 不得将未实现能力伪装为可用

## 4. 最小回归检查

执行建议：

- `python3 -m py_compile service/simple_ui.py`
- `python3 tests/test_simple_ui.py`（受限环境允许 SKIP）
- `python3 tests/test_ui_server_port_fallback.py`（受限环境允许 SKIP）

## 5. 失败处理

若验收不通过：

1. 记录失败项与复现步骤
2. 回到对应 Agent 文档修正规约
3. 修复后重新执行最小回归
