# UI_LAYOUT_SPEC

Last Updated: 2026-04-22

## 1. 目标

本规范约束 `service/simple_ui.py` 的主页面布局，确保 UI 遵循 Harness Engineering 的执行顺序：

状态可见 > 输入可控 > 主动作突出 > 结果可达。

## 2. 页面总布局

主页面采用双栏结构：

- 左栏（输入与控制）
- 右栏（状态与结果）

移动端（窄屏）自动退化为单栏顺序：先左后右。

## 3. 左栏职责（输入与控制）

### 3.1 产品身份区

必须显示：

- 产品名 `KnowledgeHarness`
- 一句话说明：导入学习资料，自动整理为结构化笔记
- 支持类型（txt/md/pdf/docx + 可选 OCR 图片）
- API 状态 Chip（仅显示“已配置/本地模式”，不回显密钥）

### 3.2 文件导入区

必须提供：

- `选择文件`
- `选择文件夹`
- `重置本次选择`
- 文件池列表（历史上传）
- 文件池辅助操作：`清空列表`、单文件`删除`

文件池每项至少显示：

- 文件名
- 扩展类型
- 大小 + 修改时间
- 勾选状态（本次将处理 / 待选择）

### 3.3 输出与选项区

必须提供：

- 输出目录输入
- 本次实际写入绝对路径提示
- 真实可用开关（不可展示未实现假开关）

对外视图默认项：

- API 协助（可选）
- 导出 docx（可选）
- 校验策略

### 3.4 主操作区

主按钮：

- `开始整理`（调试视图为`运行流水线`）

同区次级按钮：

- `API 设置`
- `进入/切换调试视图`（仅在对应开关启用时可见）

## 4. 右栏职责（状态与结果）

### 4.1 处理状态区

必须显示：

- 当前阶段标题（等待开始 / 已完成 / 已完成含告警 / 执行失败）
- 阶段说明
- 四项统计：成功数 / 失败数 / 空文本数 / 处理文件数
- 固定阶段链：读取 → 切分 → 主题粗分类 → 内容分类 → 阶段总结 → 校验 → 导出

### 4.2 告警与日志摘要区

必须显示：

- 关键告警列表（validation warnings）
- `pipeline_notes` 列表（最多展示前若干条）

### 4.3 结果摘要区

运行后显示：

- 处理统计
- 校验状态
- 导出状态
- 主题粗分类摘要（计数 + 文件级样本）
- 笔记预览

结果操作区必须包含：

- `打开结果文件`（不可用时禁用显示）
- `打开输出目录`
- 下载按钮（Markdown / Word）

## 5. 按钮层级规范

一级按钮（Primary）：

- 开始整理 / 运行流水线

二级按钮（Secondary）：

- 选择文件
- 选择文件夹
- API 设置
- 打开结果文件
- 打开输出目录

次级操作（Tertiary）：

- 重置本次选择
- 清空列表
- 删除单文件

约束：

- 一级按钮必须视觉最突出且独占一行
- 二级按钮必须弱于一级
- 次级操作必须最弱，不得与一级同权重

## 6. 可用性与错误表达

错误文本需用户可读，优先使用：

- 当前文件类型暂不支持
- 文件解析失败，请检查文件是否损坏
- 未提取到有效文本
- 未选择任何文件
- 图片/文件数量超过单次上限

禁止仅暴露内部异常名作为唯一反馈。

## 7. 非目标

本规范不覆盖：

- 核心业务流水线语义调整
- web enrichment 新能力扩展
- 云服务化改造
- 桌面壳程序交互细节（另行定义）
