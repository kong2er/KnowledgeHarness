# AGENT_B_UI_INFORMATION_ARCHITECTURE

Last Updated: 2026-04-22

Agent B（结构整理与界面信息架构 Agent）负责右栏状态与结果信息架构，保证“信息层级先于展示细节”。

## 1. 信息层级总则

优先级顺序：

1. 当前状态（是否可执行/是否完成）
2. 过程统计（成功/失败/空文本/总处理）
3. 关键告警与管线备注
4. 结果摘要与导出入口
5. 详情诊断（调试视图）

## 2. 状态区（默认展开）

必须展示：

- 阶段标题：`等待开始/已完成/已完成（含告警）/执行失败`
- 阶段说明文案
- 统计卡：成功数、失败数、空文本数、处理文件数
- 固定流程阶段链：
  - 读取资料
  - 切分
  - 主题粗分类
  - 内容分类
  - 阶段总结
  - 校验
  - 导出

## 3. 日志摘要区（默认展开）

必须展示：

- `validation.warnings`（关键告警）
- `pipeline_notes`（运行备注）

策略：

- 默认仅展示前 N 条，避免淹没主流程信息
- 内容必须为用户可读句子

## 4. 结果摘要区（运行后默认展开）

必须展示字段：

- 处理文件数
- 成功/失败/空文本
- 校验状态（is_valid + warnings 数）
- 导出状态（是否生成 md）
- 主题粗分类摘要（topic 计数 + 文件级样例）

必须包含结果可达动作：

- 打开结果文件
- 打开输出目录
- 下载 Markdown / Word

## 5. 详情区策略

- 调试视图允许展示额外诊断（如完整摘要组件）
- 对外视图禁止默认暴露过多调试细节
- 详情只能补充，不得替代主流程信息

## 6. 一致性约束

- UI 阶段名称必须与真实 pipeline 语义一致
- 不允许在 UI 中构造不存在的阶段
- Agent B 不得改写 Agent A 的输入契约，也不得改写 Agent C 的可用性验收标准
