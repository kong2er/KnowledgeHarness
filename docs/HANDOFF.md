## 当前可用交接结论

- MVP 主流程已能端到端执行并产出 `result.json`/`result.md`。
- 当前版本重点是“流程完整可跑”，不是“高质量分类准确率”。
- 下一步应先补最小可靠性，再扩展功能。

## 建议的下一步（按顺序）

1. 先补单元测试（解析/切分/分类/校验）确保回归稳定。
2. 在不改模块命名的前提下增强分类规则（词典、阈值、优先级）。
3. 接入最小 web enrichment（可开关），并保证外部信息独立分区展示。
4. 补充冲突检测策略（至少关键词冲突规则）并接入 validation。
